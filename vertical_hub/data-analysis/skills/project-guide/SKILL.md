# Project Guide（数据分析）

## 你应该怎么做

### 新对话开始时
1. 检查工作区是否有 `.phase` 文件
2. **无文件** → 用户是新来的，主动问："你好，你想分析什么数据？我来帮你从探索到报告全流程搞定。"
3. **有文件** → 读取当前阶段，按下面对应阶段的行为引导

### 阶段1：项目启动（人机协作）
你的任务是把数据分析项目从零搭建到可以自主运行。按顺序做：

1. **理解需求** → 问清楚：数据源是什么？核心表有哪些？关注什么维度？报告给谁看？→ 写 `requirements.md`
2. **试做样本** → 挑一张核心表做一次分析 → 给用户看结果 → 收集反馈 → 写 `alignment.md`
3. **建能力** → 检查现有工具够不够（SQL 查询？统计计算？报告生成？），缺什么补什么
4. **定质量标准** → 把用户反馈提炼为可检查的规则（如：必须有对比数据、必须标异常值）→ 更新 `alignment.md` 质量底线
5. **协商自进化策略** → 问用户：连续失败几次纠偏？能改什么？需不需要确认？→ 写 `evolution_policy.md`
6. **拆任务** → 按表/按维度拆分析任务 → 写 `tasks.md`

每一步做完都要展示给用户确认，再进入下一步。
详细步骤参考：`steps/01-bootstrap.md`

### 阶段2：自主执行（Capricorn 全权）
当阶段1 退出条件全部满足后：
1. 创建 executor cron（每小时领取分析任务执行）和 verifier cron（定期验证报告质量）
2. 告诉用户："设置好了，我会自主分析，有问题随时找我。"
3. 进入静默运行，不打扰用户

详细步骤参考：`steps/02-autonomous.md`

### 阶段3：人工介入（用户回来了）
用户主动发起对话时：
1. **先汇报**：分析进度 + 质量通过率 + 自动纠偏记录
2. **逐个确认**：自动修改的规则要不要保留？
3. **问新需求**：有没有新的分析方向或维度？
4. 对齐完成后回到阶段2 继续

详细步骤参考：`steps/03-intervention.md`

## 阶段1 退出条件

- [ ] `requirements.md` — 分析需求（数据源、核心表、关注维度）
- [ ] `alignment.md` — 对齐规则（分析标准、质量底线）
- [ ] `evolution_policy.md` — 自进化策略（纠偏阈值、权限）
- [ ] `tasks.md` — 分析任务列表
- [ ] 至少 1 个数据分析 tool/skill/workflow 已创建

## 阶段转换

```
1 → 2：退出条件全部满足
2 → 3：用户主动介入
3 → 2：对齐完成，回到执行
```

每次转换时更新 `.phase` 文件。详细交互范例见 `steps/example.md`（按需加载）。

## 用户犹豫"能不能放手"

当用户问"什么时候可以自动跑"、"做到什么程度够了"、"要不要再完善一下"时，加载 `steps/automation-readiness.md`。核心原则：**先跑起来，边跑边改。**
