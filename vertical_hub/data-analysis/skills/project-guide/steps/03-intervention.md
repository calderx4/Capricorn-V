# 步骤 3.4：确认或撤销自动改动

> 用户确认 bia/skill 的自动改动是否合理。

## Capricorn 应该做什么

1. **逐条展示自动改动** — bia 和 skill 的修改
2. **请求确认** — "这条规则是我加的，对不对？"
3. **处理用户决策**：
   - 确认 → 保留，标记为 `confirmed`
   - 撤销 → 回滚，标记为 `rolled_back`
   - 修改 → 按用户要求调整

## 完成标志

- [ ] 所有 bia 改动已确认或撤销
- [ ] 所有 skill 改动已确认或撤销
- [ ] changelog 状态已更新

# 步骤 3.5：审批大的改动

> 用户审批 workflow / tool 的新增。

## Capricorn 应该做什么

1. **展示待审批的代码改动** — workflow / tool 的内容
2. **解释改动原因** — 为什么要新增这个
3. **请求审批** — "这个 workflow 要不要上？需要重启才能生效"
4. **处理用户决策**：
   - 批准 → 标记生效，提醒需要重启
   - 拒绝 → 记录，不生效
   - 修改 → 按用户要求调整后再审批

## 完成标志

- [ ] 所有 workflow 改动已审批
- [ ] 所有 tool 改动已审批
- [ ] changelog 状态已更新
